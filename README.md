# 비즈니스 아이디어 검색 플랫폼

한국 시장을 타겟으로 유효한 비즈니스 아이디어를 발굴하는 프로젝트입니다.

---

## 핵심 요건

### 유효한 비즈니스 아이디어 정의

```
검색량 높음 + 서비스 없음 = 유효한 비즈니스 아이디어
```

**필수 조건:**
- 검색량: 월 1,000회 이상 (한국 시장)
- 경쟁 서비스: 없음 또는 심각하게 부족
- 사용자 니즈: "있으면 좋은 것"이 아닌 **"반드시 필요한 것"**

**선택 사항은 제외:**
- "있으면 좋지만 없어도 불편하지 않은 것"은 유효하지 않음
- 이미 다른 방법으로 해결 가능한 것은 제외
- 타겟 시장이 너무 좁은 것은 제외

### 타겟 시장

- **국가**: 한국
- **언어**: 한국어 키워드로 검색

---

## 아이디어 발굴 프로세스

### 1단계: 키워드 플래너에서 검색량 조사 (MCP Playwright 활용)

**MCP Playwright를 활용한 자동화 조사 방법:**

1. Claude에게 키워드 플래너 조사 요청
2. Claude가 MCP Playwright로 로그인 페이지 띄움:
   ```
   https://ads.google.com/aw/keywordplanner/home?ocid=7894254657&euid=1585528751&__u=5444964599&uscid=7894254657&__c=3126588793&authuser=0
   ```
3. 사용자가 직접 Google 계정 로그인 완료
4. 로그인 완료 후 Claude에게 알림
5. Claude가 화면 스냅샷 받아서 키워드 조사 작업 자동 수행

**워크플로우:**
```
[Claude] 로그인 페이지 오픈 (browser_navigate)
    ↓
[사용자] Google 로그인 수행
    ↓
[사용자] "로그인 완료" 알림
    ↓
[Claude] browser_snapshot으로 현재 화면 파악
    ↓
[Claude] 키워드 입력 및 검색량 데이터 추출
```

**설정:**
- 지역: 대한민국
- 언어: 한국어
- 검색량 필터: 월 1,000회 이상

### 2단계: 키워드 추출

1. "새 키워드 발견하기" 클릭
2. 시드 키워드 입력 (아래 카테고리 참고)
3. 검색량 높은 키워드 CSV로 내보내기

### 3단계: 경쟁 서비스 조사

각 키워드에 대해:
1. 네이버, Google에서 검색
2. App Store / Google Play에서 검색
3. 해당 니즈를 직접 해결하는 서비스가 있는지 확인

### 4단계: 필수 요건 검증

- 이 문제가 해결 안 되면 실제로 불편한가?
- 사용자가 돈을 낼 만한 문제인가?
- 대안이 없거나 대안이 매우 불편한가?

## 사용법

### 빠른 시작

이 프로젝트는 한국 시장에서 유효한 비즈니스 아이디어를 발굴하기 위한 리서치 도구입니다.

#### 1. 프로젝트 설정

```bash
# 저장소 클론
git clone https://github.com/Wanna-Be-AI-Master/BusinessIdeaSearch-ko.git
cd BusinessIdeaSearch-ko

# 의존성 설치
npm install

# 개발 서버 실행
npm run dev
```

#### 2. 키워드 플래너 로그인 (선택사항)

Google 키워드 플래너에 접속하여 검색량 데이터를 조사할 수 있습니다:

```bash
# Playwright로 키워드 플래너 열기
npm run keyword-planner-login
```

이 스크립트는:
- Chrome 브라우저를 자동으로 실행
- Google 키워드 플래너 로그인 페이지로 이동
- 사용자가 수동으로 로그인할 수 있도록 대기 (10분간 열어둠)

로그인 후 키워드를 입력하고 검색량 데이터를 수동으로 조사하세요.

#### 3. 비즈니스 아이디어 발굴하기

**조사 프로세스:**

1. **키워드 카테고리 선택**: `README.md`의 "조사할 키워드 카테고리" 참고
2. **검색량 조사**: 키워드 플래너에서 월 검색량 1,000회 이상 키워드 찾기
3. **경쟁 분석**: 네이버, Google, App Store에서 해당 키워드 검색
4. **니즈 검증**: 반드시 필요한(Must have) 서비스인지 확인
5. **결과 기록**: `keyword-research/` 폴더에 조사 결과 정리

**예시 워크플로우:**

```bash
# 1. 키워드 플래너에서 "민원 신청 앱" 검색
# 2. 월 검색량 확인 (예: 2,500회)
# 3. 네이버, Google에서 검색하여 경쟁 앱 확인
# 4. App Store에서 "민원 신청" 검색
# 5. keyword-research/새로운-키워드.md 파일 작성
```

#### 4. 조사 결과 확인

기존 조사 결과는 다음 폴더에서 확인할 수 있습니다:

- `apps/`: 분석 완료된 앱 아이디어
  - `01-kataduke/`: 정리정돈 앱 분석
  - `02-koukan-nikki/`: 교환일기 앱 분석
  - `03-setsuyaku-kondate/`: 절약 식단 앱 분석

- `keyword-research/`: 키워드별 상세 조사 결과
  - `00-종합정리-01.md`: 전체 키워드 요약
  - `78-한국앱스토어경쟁분석.md`: 한국 앱스토어 경쟁 분석
  - `79-최종결론-액션플랜.md`: 최종 결론 및 실행 계획

## 주요 기능

- **아이디어 검색**: 키워드 검색량 기반 비즈니스 아이디어 발굴
- **시장 분석**: 관련 시장 트렌드 및 경쟁 분석
- **아이디어 저장**: 관심 있는 아이디어 북마크 기능

## 기술 스택

- **Frontend**: Next.js 16, React 19, TailwindCSS 4
- **Backend**: Next.js API Routes
- **Testing**: Playwright (E2E 테스트)

## 시작하기

### 1. 의존성 설치

```bash
npm install
```

### 2. 환경 변수 설정

`.env` 파일을 생성하고 필요한 변수를 설정하세요:

```env
# App
NEXT_PUBLIC_APP_URL="http://localhost:3000"

# API Keys (필요시 추가)
```

### 3. 개발 서버 실행

```bash
npm run dev
```

[http://localhost:3000](http://localhost:3000)에서 접근할 수 있습니다.

## CI/CD

### GitHub Actions

PR 생성 및 Push 시 자동으로 CI가 실행됩니다.

| 단계 | 설명 |
|------|------|
| Install | 의존성 설치 |
| TypeCheck | 타입 체크 |
| Lint | ESLint 체크 |
| E2E Tests | Playwright 테스트 |

워크플로우 설정: `.github/workflows/ci.yml`

### 배포 트리거

main 브랜치에 푸시 시 자동으로 배포 워크플로우가 트리거됩니다.

워크플로우 설정: `.github/workflows/deploy-trigger.yml`

## 프로젝트 구조

```
src/
├── app/                    # Next.js App Router
│   ├── layout.tsx          # 루트 레이아웃
│   ├── page.tsx            # 메인 페이지
│   └── globals.css         # 글로벌 스타일
├── components/             # React 컴포넌트 (예정)
├── lib/                    # 유틸리티 함수 (예정)

e2e/                        # Playwright E2E 테스트
.github/
└── workflows/
    ├── ci.yml              # CI 워크플로우
    └── deploy-trigger.yml  # 배포 트리거 워크플로우
```
