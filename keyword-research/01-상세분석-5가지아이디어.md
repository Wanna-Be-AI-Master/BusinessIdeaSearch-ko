## 1. 作り置き (밑반찬 레시피) - 최우수 기회

### 시장 현황
- **검색량**: 1만~10만/월
- **키워드 수**: 1,429개
- **경쟁도**: 낮음
- **성장세**: ダイエット レシピ 1週間分 作り置き +900% 급성장

### 기존 경쟁사 분석

| 앱/서비스 | 특징 | 한계점 |
|----------|------|--------|
| **me:new (ミーニュー)** | 1주일 헌립 자동 생성, 220만 DL, App Store 4.3점 | 메뉴 바리에이션 부족, 작업 속도 느림 |
| **Racook (ラクック)** | 작り置き 특화, 1000+ 레시피, 조리 단계별 가이드 | 마이너 앱, 기능 제한적 |
| **クラシル** | 38,000 레시피, 동영상, 월 480엔 프리미엄 | 범용 레시피앱, 작り置き 특화 아님 |
| **つくりおき食堂** | 월 500만 PV, 누적 2.3억 PV, 15만부 서적 판매 | **블로그 형태 (앱 없음)** |

### 기존 앱의 사용자 불만점

1. **me:new 불만**
   - 메뉴 바리에이션 부족 → 장기 사용시 지루함
   - 검색/동작 속도 느림
   - 인원수 유연 설정 불가 (5인 가정에서 4인/3인 변경 어려움)
   - 헌립 입れ替え 불가능
   - 가족 공유 기능 없음 (스크린샷으로 공유해야 함)

2. **범용 레시피앱 불만**
   - "작り置き" 전용 필터 없음
   - 보존 기간/냉장/냉동 정보 부족
   - 여러 요리 동시 조리 시 단계별 가이드 없음

### 차별화 전략

```
[핵심 포지셔닝]
"1주일 작り置き 전문 앱"
- 블로그(つくりおき食堂)의 앱 버전을 노린다
- me:new의 불만점을 해결한다
```

### 개발 시 가치 제안

#### 핵심 기능
1. **AI 작り置き 플래너**
   - 1주일치 밑반찬 자동 제안
   - 인원수 유동 설정 (오늘은 4인, 내일은 3인)
   - 냉장고 재료 기반 추천

2. **동시 조리 타임라인**
   - 3-4가지 밑반찬을 동시에 만들 때 최적 순서 제안
   - "지금 ○○은 끓이는 중, △△를 자르세요" 실시간 가이드
   - Racook 기능 + 개선

3. **보존 관리**
   - 냉장/냉동 보존 기간 알림
   - "이번 주 먹어야 할 것" 우선순위 표시
   - 사진 기록 (Before/After)

4. **가족 공유**
   - 실시간 헌립 공유 (파트너가 대신 요리 가능)
   - 장보기 리스트 공동 편집

5. **건강 식단 연계**
   - 糖尿病 레시피, ダイエット 레시피 태그
   - 영양소 자동 계산

#### 수익화 모델
| 모델 | 가격 | 내용 |
|------|------|------|
| 무료 | 0엔 | 기본 레시피 200개, 1주일 플래너 |
| 프리미엄 | 월 480엔 | 전체 레시피, AI 추천, 가족 공유, 광고 제거 |
| 연간 플랜 | 연 3,800엔 | 월 대비 34% 할인 |

## 2. 節約 + 献立 (식비 절약 식단) - 최고 유망

### 시장 현황
- **節約 검색량**: 10만~100만/월 (+900% 성장)
- **献立 검색량**: 1만~10만/월
- **키워드 수**: 2,654개
- **경쟁도**: 낮음

### 시장 배경 (2024-2025)

| 지표 | 수치 | 출처 |
|------|------|------|
| 1인 가구 평균 식비 | 월 43,941엔 | 총무성 가계조사 2024 |
| 전년 대비 식비 상승 | +4.5% | - |
| 식품 물가지수 | 122.5 (2020=100) | - |
| 대학생 평균 식비 | 월 26,110엔 | - |

**핵심 인사이트**:
- 일본 식품 가격 고공행진 → 절약 니즈 급증
- 1인 가구는 재료 소분 어려움 → 식비 비효율

### 기존 경쟁사 분석

| 앱 | 특징 | 한계점 |
|----|------|--------|
| **me:new** | 특매 정보 연동, 장보기 리스트 | 절약 "전문" 아님 |
| **クラシル** | 조리시간/비용 표시, 특매 전단지 | 절약 특화 아님 |
| **クックパッド** | 냉장고 재료 검색 | 유저 레시피 품질 불균일 |
| **楽天レシピ** | 포인트 적립 | UI 복잡, 레시피 품질 낮음 |

### 기존 앱의 사용자 불만점

1. **절약 효과 불명확**
   - "이 식단으로 얼마나 절약되는지" 시각화 없음
   - 예산 설정 → 식단 자동 조정 기능 없음

2. **1인분 레시피 부족**
   - 대부분 2-4인분 기준
   - 1인 가구에게 비효율적

3. **특매 정보 활용 한계**
   - 전단지 보기만 가능, 식단 자동 연동 없음

### 차별화 전략

```
[핵심 포지셔닝]
"월 식비 2만엔으로 건강하게 먹는 앱"
- 예산 먼저 설정 → 식단 역산
- 1인 가구 + 젊은 층 타겟
```

### 개발 시 가치 제안

#### 핵심 기능

1. **예산 우선 식단 플래너**
   - "이번 주 예산 5,000엔" 입력
   - AI가 예산 내 최적 식단 자동 생성
   - 영양 밸런스 유지

2. **절약 효과 시각화**
   - "이번 달 ○○엔 절약!" 대시보드
   - 월별 식비 트렌드 그래프
   - 평균 대비 절약률 표시

3. **특매 연동 최적화**
   - 근처 슈퍼 특매 정보 수집
   - 특매 재료 기반 식단 자동 조정
   - "오늘 계란이 특가! 이 레시피 추천"

4. **1인분 레시피 특화**
   - 모든 레시피 1인분 기준
   - "소분 보관법" 가이드
   - 재료 사용률 100% 목표

5. **장보기 최적화**
   - 슈퍼별 가격 비교
   - 최저가 루트 제안
   - 과잉 구매 방지 알림

#### 수익화 모델

| 모델 | 가격 | 내용 |
|------|------|------|
| 무료 | 0엔 | 기본 식단, 장보기 리스트 |
| 프리미엄 | 월 380엔 | AI 최적화, 특매 연동, 절약 분석 |

### 기존 경쟁사 분석

| 앱/서비스 | MAU/규모 | 특징 | 한계점 |
|----------|----------|------|--------|
| **ママリ** | 600만 MAU, 300만 회원 | Q&A, 커뮤니티, 월 400엔 프리미엄 | "양립" 특화 아님 |

### 기존 앱의 사용자 불만점

1. **"양립" 전용 기능 없음**
   - 기존 앱은 "육아 기록" 중심
   - 일과 육아 스케줄 통합 관리 없음

2. **긴급 상황 대응 미흡**
   - 아이 아플 때 → 베이비시터 급구 연동 없음
   - 회사에 연락할 템플릿 없음

3. **커리어 지원 부족**
   - 복직 후 커리어 플랜 가이드 없음
   - 시간제 근무 최적화 도구 없음

### 차별화 전략

```
[핵심 포지셔닝]
"워킹맘/대디의 일-육아 양립 전용 앱"
- 기존 육아앱 + 캘린더/업무 도구 융합
- 92%가 느끼는 "양립 곤란" 직접 해결
```

#### 핵심 기능

1. **통합 캘린더**
   - 업무 일정 + 아이 일정 (예방접종, 등하원) 한눈에
   - 부모 간 일정 공유
   - 충돌 알림 ("다음 주 출장 vs 예방접종")

2. **긴급 상황 SOS**
   - 아이 아플 때 체크리스트
   - 병원 예약 연동
   - 베이비시터/임시보육 매칭
   - 회사 연락 템플릿 ("자녀 발열로 재택 전환")

3. **시간 최적화**
   - "○분 짜투리 시간" 활용 팁
   - 집중 업무 시간대 제안
   - 가사 분담 트래커

4. **커리어 플래닝**
   - 복직 단계별 가이드
   - 시간제 근무 전환 팁
   - 롤모델 인터뷰/사례

5. **커뮤니티**
   - 같은 상황의 워킹맘/대디 연결
   - 익명 Q&A
   - 지역별 정보 공유

## 5. 交換日記 (교환일기) - 급성장 시장

### 시장 현황
- **검색량**: 1만~10만/월 (+900% 성장)
- **경쟁도**: 낮음

### 기존 경쟁사 분석

| 앱 | 특징 | 한계점 |
|----|------|--------|
| **記憶の木** | 커플 전용, 질문 응답형, 나무 성장 | 커플만 가능, 친구/가족 X |
| **家族ダイアリー** | 가족 공유, ChatGPT 댓글, 고양이 UI | 통지 불안정, 기능 제한적 |
| **Shanfy** | 그룹 교환일기, 일일 질문 | **대규모 사양 변경으로 유저 이탈** |
| **Waffle** | 40종 커버, 프롬프트 기능 | 차별점 부족 |

### 기존 앱의 사용자 불만점

1. **치명적 신뢰 문제 (Shanfy)**
   - 갑작스러운 사양 변경 → 3년치 일기 삭제
   - 백업에 1,800엔 요구 → 유저 분노
   - "두 번 다시 사용 안 함" 리뷰 다수

2. **통지 불안정**
   - 상대방 글 알림 안 옴 → 교환일기 단절
   - 실시간성 상실
