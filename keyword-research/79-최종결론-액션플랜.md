## 4. 기술 스택 정리

### 모바일 앱 (片付け, 交換日記)

```
Frontend: React Native
Backend:  Supabase (PostgreSQL, Auth, Storage)
Push:     Firebase Cloud Messaging
AI:       OpenAI API
Analytics: Mixpanel / Amplitude
```

### 웹앱 (節約献立)

```
Frontend: Next.js (Kimatch 포크)
Backend:  Next.js API Routes + Prisma
DB:       PostgreSQL (Supabase)
AI:       OpenAI API
PWA:      오프라인 장보기 리스트
```

## 5. 각 앱 핵심 차별점

### 片付け (정리정돈)

| USP | 설명 |
|-----|------|
| AI 버림 분석 | "충동구매 의류를 가장 많이 버렸습니다" |
| 환경 가치 | CO2 절감량 계산 |
| メルカリ 연동 | 버리기 전 판매 추천 |

### 交換日記 (교환일기)

| USP | 설명 |
|-----|------|
| 데이터 영구 보증 | "절대 삭제하지 않습니다" |
| 다중 관계 지원 | 커플+가족+친구 통합 |
| 100% 푸시 알림 | 상대방 글 놓치지 않음 |

### 節約献立 (식비 절약)

| USP | 설명 |
|-----|------|
| 예산 우선 플래닝 | 예산 입력 → 식단 자동 생성 |
| 절약 효과 시각화 | "이번 달 12,500엔 절약!" |
| 동시 조리 타임라인 | 여러 요리 동시 조리 가이드 |

## 7. 액션 플랜

### Phase 1: 1순위 앱 개발 (片付け)

```
Week 1-2: 프로젝트 셋업
├── React Native 프로젝트 생성
├── Supabase 연동
└── 기본 Auth 구현

Week 3-4: 핵심 기능
├── 일일 트래커
├── 카테고리 분류
├── 연속 일수 카운터
└── 대시보드

Week 5-6: 확장 + 출시
├── 게이미피케이션
├── 테스트
└── 스토어 출시
```

### Phase 2: 2순위 앱 개발 (交換日記)

```
Week 7-8: 프로젝트 셋업
├── React Native 프로젝트 생성
├── 실시간 DB 설계
└── 푸시 알림 구현

Week 9-11: 핵심 기능
├── 일기장 생성/초대
├── 일기 작성
├── 읽음 확인
└── 자동 백업

Week 12: 출시
├── 테스트
├── "데이터 영구 보증" 마케팅
└── Shanfy 이탈자 타겟 ASO
```

### Phase 3: 3순위 웹앱 (節約献立)

```
Week 13-14: Kimatch 포크
├── 프로젝트 복사
├── DB 스키마 재설계
└── 불필요 코드 제거

Week 15-18: 핵심 기능
├── 예산 설정
├── 식단 플래너
├── 레시피 CRUD
├── 장보기 리스트
└── PWA

Week 19-20: SEO + 출시
├── 블로그 콘텐츠
├── 구조화된 데이터
└── 정식 출시
```
